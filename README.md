ETL & Power BI Pipeline for Automation of Crypto in Azure Environment. This provides the entire process flow from automated end-to-end data engineering by automating the ingestion of crypto data, and then storing, transforming, and eventually displaying in near real-time. Medallion flow for architecture: Bronze is unstructured JSON files as received from CoinGecko API and stored in Microsoft Azure blob storage.

The transition of data from JSON into an Azure SQL Database happened via Microsoft Azure Data Factory, all clean. For daily high throughput report generating purposes, optimised SQL views provide market data for up-to-date snapshots of the marketplace. The technology stack that was used is as follows: CoinGecko API for the source of the data; Azure Logic Apps (triggered daily) for the orchestration and ingestion of the data; Azure Data Lake Storage to store the raw data from the MongoDB Azure Blob storage system post ETL or integrated Into the data lake; Azure Data Factory (ADF) to facilitate ETL/integration; Azure SQL Databases were used as the data warehouse (Silver - Azure SQL Database, Gold - Azure SQL Database).

Visual Evaluation - Details of Royalty Information Technology: Royalty Processing - New Workflow for Workflow to Process Data into Process: The Azure Logic Apps have a daily job that will utilize Azure Logic Apps to get data from external systems (HTTP GET) via HTTP GET to be stored in Azure Blob Storage with the dynamic time stamp, in the following manner: crypto_##TIMESTAMP##.json.
The Azure Data Factory manages data migration process to the SQL database using the ETL / transformation (Silver Layer) process. To process the batch of daily JSON files, the Copy Activity utilizes the wildcard path. With respect to the Schema Mapping, the nested values from JSON are accurately mapped to columns in a SQL table based on the required transformations from JSON to ISO/Decimal to ensure finanical accuracy.

In the Gold Layer of Data Modeling, dashboards were dramatically enhanced by utilizing a gold view in SQL. Instead of populating millions of historical records into the "Current Prices" card, we utilize a view that will return the current/recent prices of all corresponding crypto prices from the CryptoPrices table. The SQL to accomplish that is as follows: SQL CREATE VIEW vw_gold_aktualne_ceny AS SELECT name, price_usd, last_updated FROM CryptoPrices WHERE last_updated = (SELECT MAX(last_updated) FROM CryptoPrices) Professional Visualizations (Power BI) were utilized in order to: display, visually present or otherwise provide actionable insight through the following:

Button Slicers For Using Coins Together: Allows Coin Swapping Easily and Quickly between Bitcoin, Ethereum and Solana with No Hassle. Format Complexity: Create a UI in Dark Mode with Conditional Colour Levels in Bar Graph Representations to Show The Very Highest Token by Bar Height. AI/ML Automation Tools to Use with Data: Uses Smart Narratives That Describe the Market Trends Found in Data for Simplification. DAX Examples for Custom Measures: Logic applied to Currency Formatting Properly to Not Give Facility "Auto-Scaled" Values (e.g. number of Thousands as "K") as an Actual Value.

The timestamps provided by the API were all iso formatted (ex: 2026-03-30T12:00:00Z) and were not usable in standard European locale formats which caused numerous technical issues.

To address our need for converting all date-time to consistent, Eastern Time (ET), we added to the Power Query a transformation that changed the Locale of our import process to English (United States) for the last_updated column. DAX Precision Formatting of Currency - To enable precise values for USD on the dashboard, with no forced rounding, and without "K" at the end of USD amount, I developed a custom DAX measure, that will calculate price display as: Price Display = FORMAT(SUM(vw_gold_aktualne_ceny[price_usd])," #,##0.00 $") End to end automation - Completely automated from end-to-end, the entire data flow is automatic. Data flow is triggered at 7:00 AM by Azure Logic Apps & ADF.

At 7:15 AM, Azure SQL cleanses and saves data. The Power BI Cloud is refreshed at 8:00 AM as part of the scheduled schedule to obtain gold data updates. The end result is that when the user wakes up in the morning, they will have a completely up-to-date dashboard with no clicks required.
